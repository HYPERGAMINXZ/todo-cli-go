# Code of Conduct

## Our Pledge

We are committed to providing a friendly, safe, and welcoming environment for all contributors, regardless of experience level, gender identity and expression, sexual orientation, disability, personal appearance, body size, race, ethnicity, age, religion, nationality, or other similar characteristic.

## Our Standards

### Positive Behavior ✅

- **Be respectful** - Value different viewpoints and experiences
- **Be constructive** - Provide helpful, actionable feedback
- **Be collaborative** - Work together towards common goals
- **Be patient** - Remember everyone was a beginner once
- **Be inclusive** - Welcome and support people of all backgrounds
- **Give credit** - Acknowledge others' contributions

### Unacceptable Behavior ❌

- Harassment, trolling, or insulting/derogatory comments
- Personal or political attacks
- Public or private harassment
- Publishing others' private information without permission
- Sexual language, imagery, or unwelcome attention
- Other conduct which could reasonably be considered inappropriate

## Scope

This Code of Conduct applies to:

- Project repositories and issues
- Pull requests and code reviews
- Project communication channels (discussions, comments)
- Public spaces when representing the project
- Private communications related to project business

## Enforcement

### Reporting

If you experience or witness unacceptable behavior, please report it by:

- Opening a confidential issue (if supported)
- Emailing: conduct@yourdomain.com (replace with actual contact)

Include:

- Your contact information
- Names of individuals involved
- Description of the incident
- Any supporting evidence (links, screenshots)
- Whether you wish to remain anonymous

### Actions

Project maintainers will:

1. Review and investigate all complaints promptly and fairly
2. Respect the privacy and security of the reporter
3. Take appropriate action based on severity

Possible responses include:

- **Warning** - Private written warning with clarity about violation
- **Temporary Ban** - Temporary suspension from project interaction
- **Permanent Ban** - Permanent removal from project participation

## Maintainer Responsibilities

Project maintainers are responsible for:

- Clarifying standards of acceptable behavior
- Taking appropriate and fair corrective action
- Removing, editing, or rejecting contributions that violate this Code
- Banning temporarily or permanently any contributor for behaviors deemed inappropriate
